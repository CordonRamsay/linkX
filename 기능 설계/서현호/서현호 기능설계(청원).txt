청원 게시판

**필요한 화면**

1.청원글 목록화면
2.청원 상세보기 화면
3.청원 글 작성화면

(글쓰기 화면이라 단순히 적지 말고 작성자 제외하고 수정이 불가능하게끔)
(DB유저테이블의 액티브 컬럼-가입 승인 여부)
(DB유저테이블의 어쏠리티 컬럼 - 관리자, 일반 구분)

#청원글 목록 화면
-해당 게시글 리스트{청원글번호, 청원분야,청원제목,청원인(이름은 성만 표시 이름은 마스킹 처리),청원 기간,동의자수,마감일}표시
-검색기능(청원 제목으로 검색 가능)
-페이징 기능(1페이지당 글 10개 정렬, 각 페이지 번호로 페이지 이동, 이전, 다음 페이지 이동)
-정렬기능(글 작성 순서, 동의자수로 정렬가능), 순서 지정 기능(오름차순, 내림차순) >드롭다운 버튼으로 지정
-청원글 작성 하기 버튼, 클릭시 청원 글 작성화면으로 이동
-해당 글(행 선택) 클릭 시 상세보기 화면으로 이동
-청원 글의 상단의 주의사항 표시(청원글은 작성자의 수정, 삭제가 불가함을 안내)
-글의 상단의 청원 분야 별로 버튼 표시, 해당 버튼은 and로 해당하는 분야 선택할 시 해당 하는 분야 글만 보게 가능
-유저테이블의 액티브 컬럼을 참조해서 가입 승인여부를 확인후, 미승인 유저일 경우 글제목만 볼 수 있고 상세보기 불가능
/승인 유저일 경우 글제목과 상세보기 가능

#청원 상세보기 화면
-상단에 해당 청원 글의 청원 분야, 청원 기간, 동의자 수 , 청원글 게시일 표로 표시
-해당 청원 글의 청원인,청원 글을 쓰게된 취지, 청원하고자 하는 내용 표시 (줄로 나눠서)
-하단의 목록 버튼, 클릭시 청원글 목록 화면으로 이동
-하단의 동의 하기 버튼, 클릭시 동의를 다시 취소 할 수 없을 을 알리는 팝업 표시, 청원글의 동의자수 증가
-유저 테이블의 어쏠리티 컬럼을 참조하여 관리자 여부 확인, 관리자일 경우 하단의 수정, 삭제 버튼 활성화
-삭제 버튼클릭시 데이터 삭제
-수정 버튼 클릭시 청원글 작성화면으로 넘어가 기존의 입력된 내용들을 불러오고 수정한후 작성하기 버튼
클릭시 수정된 내용으로 청원 글 등록

#청원 글 작성화면
-청원글의 분야를 드롭다운 버튼 형식으로 기존의 분야들중 선택하게끔
-청원 제목, 청원 취지, 내용 작성 칸 입력
-분야,제목,취지,내용 모두 입력해야 글 작성(등록)가능
-글 작성 버튼을 클릭시 수정과 삭제가 불가함과 맞지 않는 청원 글 작성시 관리자의 의해 삭제될 수 있음을 알리는 안내
팝업 표시,팝업의 확인 버튼 클릭,글 등록 및 청원 상세 보기 화면으로 이동
